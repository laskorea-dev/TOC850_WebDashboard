# 🌐 대시보드 웹 클라우드 무료 배포 가이드 (Vercel 배포 가이드)

본 문서는 개발 완료된 **TOC B2B 대시보드**를 서버 비용 없이 **100% 무료**로 클라우드에 배포하여, 고객사 컴퓨터, 모바일 핸드폰, 태블릿 등 전 세계 어디서나 접속할 수 있도록 만드는 가이드입니다.

---

## 💎 왜 Vercel(버셀)을 사용하나요?

Vercel은 현재 전 세계 개발자들이 React/Vite 앱을 호스팅할 때 가장 많이 쓰는 최첨단 서버리스 웹 호스팅 플랫폼입니다.

1. **완전 무료 (Hobby Tier)**: 개인 프로젝트 및 스타트업 검증용으로 **평생 무료**로 쓸 수 있습니다.
2. **트래픽 및 성능 최고**: 전 세계 초고속 CDN 분산 처리를 기본 지원하여 모바일이나 회사 PC 어디서나 0.1초 만에 로드됩니다.
3. **Supabase 연동 최적화**: 서버(Backend) 없이 프론트엔드가 수파베이스 클라우드에 직접 붙기 때문에, Vercel의 무료 정적 호스팅만으로 모든 관제 기능이 무결하게 작동합니다.
4. **HTTPS 보안 인증서 자동 발급**: `https://...` 보안 프로토콜과 무료 도메인(`.vercel.app`)이 자동으로 제공됩니다.

## 🚀 배포 방법 1: GitHub 연동 방식 (가장 권장 ⭐️)

코드를 깃허브(GitHub)에 올려두고 Vercel과 클릭 한 번으로 연결해두면, **앞으로 코드가 수정될 때마다 실시간으로 웹페이지가 자동 업데이트**되는 최첨단 방식입니다.

### [1단계] GitHub에 소스코드 올리기
1. [GitHub 공식 사이트](https://github.com/)에 가입하고 로그인합니다.
2. 우측 상단 `[New]` 단추를 눌러 새 저장소(Repository)를 만듭니다.
   - 이름: `toc-b2b-dashboard`
   - 공개 여부: **Private(비공개)**로 설정합니다. (Supabase API 키 등 보안 정보 보호용)
3. 해당 Git 저장소에 현재 `dashboard` 폴더 내의 소스코드 전체를 업로드합니다.

### [2단계] Vercel 계정 생성 및 깃허브 연동
1. [Vercel 공식 사이트](https://vercel.com/)에 접속합니다.
2. **`[Sign Up]` ➔ `[Continue with GitHub]`**를 클릭하여 깃허브 계정으로 간편 가입 및 연동합니다.
3. 가입 후 대시보드 화면에서 **`[Add New...]` ➔ `[Project]`** 단추를 누릅니다.

### [3단계] 프로젝트 빌드 및 배포
1. 깃허브 저장소 목록 중 방금 만든 `toc-b2b-dashboard` 저장소를 찾아 **`[Import]`**를 누릅니다.
2. **Configure Project** 화면이 나타나면 아래 두가지만 체크합니다:
   - **Root Directory**: `dashboard` (현재 프로젝트 루트 내에 `dashboard` 서브 폴더에 웹 소스가 들어 있으므로 지정해 주어야 합니다.)
   - **Framework Preset**: `Vite` (자동 감지되나, 안 되어 있다면 Vite로 선택)
3. 하단의 **`[Deploy]`** 단추를 누릅니다.
4. 약 1~2분 뒤 빌드가 완료되면 폭죽 효과와 함께 **전 세계 어디서나 접근 가능한 고유 주소**가 발급됩니다! (예: `https://toc-b2b-dashboard.vercel.app`)

## ⚡ 배포 방법 2: Vercel CLI 명령어 방식 (초고속 간편 배포)

GitHub 가입 없이 컴퓨터 터미널 창에서 명령어 한 줄로 즉시 전 세계에 배포하는 방식입니다.

### [1단계] Vercel CLI 도구 실행
1. 윈도우 파워쉘(PowerShell) 또는 CMD를 열고 `dashboard` 폴더로 이동합니다.
2. 아래 명령어를 입력하여 Vercel CLI 설치 및 로그인을 진행합니다.
   ```bash
   npx vercel login
   ```
   - 브라우저 창이 열리면 로그인을 마칩니다.

### [2단계] 즉시 웹 배포 실행
1. 로그인 완료 후, `dashboard` 폴더 위치에서 아래 명령어를 입력합니다.
   ```bash
   npx vercel
   ```
   - 터미널 창에 물어보는 질문들에 대해 엔터(Enter) 키만 쳐서 넘어가면 됩니다. (기본값 설정 적용)
     - *Set up and deploy?* `y`
     - *Which scope?* (내 이름 선택)
     - *Link to existing project?* `n`
     - *What's your project's name?* `toc-b2b-dashboard` (엔터)
     - *In which directory is your code located?* `./` (엔터)
2. 약 30초 뒤 완료되면 개발용 임시 주소(Preview URL)가 즉시 출력됩니다.
3. 최종적으로 **실제 운용(Production) 서버**로 확정 배포하려면 다음 명령어를 치면 끝납니다:
   ```bash
   npx vercel --prod
   ```
   - 웹 서버에 영구 릴리즈되며 최종 모바일 접속용 퍼블릭 주소가 터미널창에 표시됩니다!

## 🤝 B2B 멀티테넌시: 1개의 GitHub 코드로 수십 명의 고객사 호스팅하기

"새로운 고객사가 생길 때마다 깃허브 저장소를 새로 복사해야 할까요?"
**정답은 "아닙니다!" 1개의 GitHub 저장소만으로 수천 개의 독립된 고객사 대시보드를 완전히 분리하여 안전하게 호스팅할 수 있습니다.**

현재 대시보드 소스코드는 **Vite 환경 변수**(`import.meta.env`)를 사용해 동적으로 Supabase 주소와 키를 주입받도록 완벽히 고도화되어 있습니다.

### ⚙️ 멀티테넌시 구성 및 보안 분리 절차
1. **깃허브 저장소는 딱 1개**만 유지합니다. (나중에 기능 추가나 디자인 개선을 했을 때, 단 한 번의 커밋으로 모든 고객사 사이트에 일괄 자동 반영하기 위함입니다.)
2. Vercel에서 새로운 고객사가 추가될 때마다 **`Add Project`** 버튼을 누르고 **동일한 깃허브 저장소**를 연결합니다.
3. 배포 설정 화면의 **`Environment Variables` (환경 변수)** 탭을 펼치고, 해당 고객사만을 위해 새로 발급된 Supabase 접속 정보를 각각 다르게 기입해 줍니다:
   - **이름(Key)**: `VITE_SUPABASE_URL`  ➔  **값(Value)**: `https://고객사별주소.supabase.co/rest/v1/`
   - **이름(Key)**: `VITE_SUPABASE_KEY`  ➔  **값(Value)**: `고객사별 API Key 값`
4. 각 Vercel 프로젝트의 이름(도메인 주소)을 해당 고객사명에 맞춤형으로 다르게 부여합니다:
   - A사 전용 모바일 관제판: `https://company-a-toc.vercel.app` (A사 Supabase 연결)
   - B사 전용 모바일 관제판: `https://company-b-toc.vercel.app` (B사 Supabase 연결)

이 방법을 사용하면 **완벽한 데이터 물리적 보안 격리(Multi-Tenancy Isolation)**가 실현되면서도, 소스코드를 여기저기 복사-붙여넣기 할 필요가 없어 앞으로 대시보드를 관리하고 유지보수하기가 압도적으로 쉬워집니다.

## 📱 스마트폰 및 외부 PC에서 확인하기

- Vercel에서 제공받은 고유 도메인 주소(예: `https://toc-b2b-dashboard.vercel.app`)를 **카카오톡, 이메일, 문자** 등으로 나 자신과 고객에게 공유합니다.
- 모바일 핸드폰 브라우저(Safari, Chrome 등) 및 사무실 PC 등 기기를 불문하고 해당 주소를 치면 **실시간으로 수파베이스 DB에 누적되고 있는 최신 TOC 농도와 추이 꺾은선 차트**가 즉각 반응하며 열립니다!
