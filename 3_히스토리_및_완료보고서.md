# 📘 TOC B2B 멀티테넌시 계측 모니터링 시스템 최종 완료보고서 & 사용자 매뉴얼

본 문서는 개발이 완료된 다중 계측기 동기화 및 B2B 멀티테넌시(고객사별 독립 데이터 격리) 대시보드 시스템의 운용, 환경 설정 및 라이브 배포를 위한 종합 사용자 가이드북입니다.

---

## 1. B2B 멀티테넌시 최종 개발 주요 성과

여러 대의 계측기를 동시에 돌리고 다수의 회사(고객사)에 대시보드 모니터링 서비스를 유료 구독이나 서버 비용 없이 제공해야 하는 비즈니스 제약 조건을 해결하기 위해 다음의 혁신적 설계를 반영하였습니다.
1. **완벽한 물리적 보안 격리 (0원 무서버 B2B)**:
   * 고객사(회사)마다 별도의 구글 스프레드시트를 개설합니다. (예: A사 시트, B사 시트)
   * React 대시보드 정적 빌드 결과물을 고객사별로 각각 독립 배포(Vercel, Netlify 정적 주소 무제한 무료 개설 지원)하여 고유 링크를 제공합니다. (예: `company-a.vercel.app`, `company-b.vercel.app`)
   * 각 회사 대시보드에는 본인 회사의 구글 시트 링크만 맵핑되므로 타사 데이터가 노출되거나 섞일 위험이 0%입니다.
2. **다중 기기 식별자 (`Device_ID`) 탑재**:
   * 한 고객사 공장 내에서 여러 대의 계측기를 가동하는 경우에도 충돌 없이 하나의 시트에 통합 누적하기 위해, 데이터 행마다 `Device_ID`를 강제 매립 전송하도록 규격을 고도화하였습니다.
   * 대시보드 웹앱 내에는 고유 기기 ID를 중복 제거하여 드롭다운으로 자동 완성하는 **장비 필터 셀렉트 박스**가 내장되어, 실시간 요약 카드, Recharts 시간별 추이 차트, 바 차트, 그리고 하단 이력 테이블이 클릭 즉시 해당 장비의 데이터로 연쇄 정렬 및 렌더링을 시작합니다.
3. **중앙 제어형 설정 분리 (`uploader_config.json`)**:
   * 소스 코드를 건드리지 않고 SQLite DB 파일 경로, 업로드 목적지 구글 시트 이름, 가동 주기(초), 장비 식별자 ID를 파일에서 손쉽게 셋업할 수 있도록 설계했습니다.
   * GUI 창에 직접 텍스트 박스와 `장비ID 저장`, `이름 저장` 단추를 제공하여 프로그램 창상에서 즉시 설정을 실시간 변경하고 동기화할 수 있습니다.

## 2. 세부 구동 및 현장 배포 가이드라인

### [1단계] 계측기 장비 PC 셋업 (다중 기기 등록)
공장에 있는 여러 대의 계측기 PC에 각각 업로더를 셋업하는 방법입니다.

1. 계측기 A 컴퓨터에 `db_upload_and_dashboard` 폴더를 복사하고 **`run_uploader.bat`** 파일을 켭니다.
2. 프로그램 창이 뜨면 다음 설정을 지정합니다.
   * **연동 SQLite DB 파일 경로**: `[찾아보기...]`를 눌러 해당 PC 내의 계측 DB 파일(`toc_db.db`)을 지정합니다.
   * **구글 스프레드시트 이름**: 이 컴퓨터가 데이터를 업로드해야 할 **고객사의 스프레드시트 이름**을 타이핑하고 `[이름 저장]`을 누릅니다.
     * 예: A사 현장 기기라면 `TOC_Dashboard_Company_A` 입력
   * **계측 장비 고유 ID 식별자**: 이 장비의 고유한 이름(예: `DEVICE_A1`, `DEVICE_A2`)을 입력하고 `[장비ID 저장]` 단추를 누릅니다.
3. 타이머가 15분 주기로 백그라운드 스레드에서 자동 구동하기 시작하며 데이터에 장비 ID를 매립하여 구글 시트로 안정적으로 동기화합니다.
   * *로컬 테스트 시*: 구글 API 인증 키(`credentials.json`)가 없으면 기본적으로 시뮬레이션 모드(`--mock`)로 기동하여 웹 대시보드 퍼블릭 CSV 폴더(`dashboard/public/mock_google_sheet.csv`)에 데이터를 자동으로 차곡차곡 모아줍니다.

### [2단계] 실제 라이브 구글 시트 (스프레드시트) API 세팅법
실제 클라우드로 실시간 B2B 전송을 개설하기 위한 구글 클라우드 콘솔 설정입니다.

1. **Google Cloud Console 가입 및 프로젝트 생성**:
   * [Google Cloud Console](https://console.cloud.google.com/)에 구글 계정으로 로그인합니다.
   * 상단 프로젝트 선택창에서 `새 프로젝트`를 생성합니다 (예: `TOC-B2B-Monitor`).
2. **Google API 서비스 활성화**:
   * 검색창에 **Google Sheets API**를 검색하고 들어가 `사용(Enable)`을 클릭합니다.
   * 마찬가지로 **Google Drive API**도 검색하여 `사용(Enable)`을 클릭합니다.
3. **서비스 계정(Service Account) 및 JSON 키 발급**:
   * `API 및 서비스` -> `사용자 인증 정보`로 이동합니다.
   * 상단 `사용자 인증 정보 만들기` -> `서비스 계정(Service Account)`을 생성합니다 (예: `toc-b2b-uploader`).
   * 생성된 서비스 계정 이메일(예: `toc-b2b-uploader@...iam.gserviceaccount.com`)을 안전하게 복사해 둡니다.
   * 이메일을 클릭하여 상세 페이지 진입 후, `키(Keys)` 탭 -> `키 추가` -> `새 키 만들기` -> **JSON**을 선택해 다운로드합니다.
4. **인증서 보관**:
   * 다운로드받은 파일명을 **`credentials.json`**으로 변경합니다.
   * 이 파일을 `d:\antigravity\db_upload_and_dashboard\` 폴더(즉, `gui_uploader.py`가 있는 폴더) 내에 넣어줍니다.
5. **고객사별 구글 시트 생성 및 공유**:
   * 구글 드라이브에서 신규 구글 스프레드시트를 만들고 파일명을 고객사 시트명과 동일하게 개설합니다. (예: `TOC_Dashboard_Company_A`)
   * 생성된 서비스 계정 이메일 주소를 복사해, 구글 시트의 **공유** 단추를 누른 뒤 **편집자(Editor)** 권한으로 초대합니다.
6. **업로더 프로그램 실제 모드로 전환**:
   * `run_uploader.bat` 파일을 우클릭하고 `편집`을 선택합니다.
   * `python gui_uploader.py --mock` 명령어 맨 앞자리에 `::` 주석 처리를 하고, 아래 실제 모드 실행 코드의 주석을 해제합니다.
   ```bat
   :: [테스트 모드]
   :: python gui_uploader.py --mock

   :: [실제 연동 모드]
   python gui_uploader.py
   ```
   * 변경 사항을 저장하고 닫습니다. 이후 켜지는 프로그램은 실제 구글 시트에 데이터를 즉시 실시간 동기화하기 시작합니다.

### [3단계] 웹 대시보드 배포 및 B2B 고객 격리 연동
1. **구글 시트 웹에 게시 처리 (가장 강력한 API 키 없는 연동 방식)**:
   * 고객사 구글 시트 메뉴 상단에서 `파일` -> `공유` -> `웹에 게시`를 차례로 누릅니다.
   * 링크 탭에서 [전체 문서] 우측의 선택창을 **[쉼표로 구분된 값(.csv)]**으로 지정하고 `게시` 단추를 누릅니다.
   * 발급되는 `https://docs.google.com/spreadsheets/.../pub?output=csv` 형식의 주소를 안전하게 복사합니다.
2. **대시보드 코드에 대입**:
   * `dashboard/src/App.jsx` 파일을 텍스트 에디터로 엽니다.
   * 최상단 부근에 있는 `const GOOGLE_SHEETS_CSV_URL = "";` 변수 큰따옴표 사이에 복사한 링크 주소를 삽입하고 저장합니다.
   ```javascript
   const GOOGLE_SHEETS_CSV_URL = "https://docs.google.com/spreadsheets/d/e/2PACX-.../pub?output=csv";
   ```
3. **Vercel 무료 B2B 다중 배포**:
   * `dashboard` 폴더 내에서 터미널을 열고 `npm run build`를 실행하여 컴파일된 정적 산출물 폴더인 **`dist`** 폴더를 획득합니다.
   * [Vercel](https://vercel.com/) 에 가입하여 대시보드 프로젝트를 업로드하고 배포 주소를 발급받습니다.
   * 다른 회사에 새로 대시보드를 제공해야 할 때는 새로운 구글 시트를 만들어 웹 게시 주소를 `App.jsx`에 맵핑하고, Vercel에서 새로운 프로젝트 이름(예: `company-b-toc.vercel.app`)으로 정적 배포하면 되며, **이 모든 프로젝트 수십 개 배포가 Vercel 무료 티어 내에서 영구 공짜로 완벽한 데이터 보안 격리를 이루며 안전하게 동작합니다!**
