# 프로젝트 개발 타임라인 (TOC-850 WebDashboard)

이 문서는 TOC-850 온라인 계측 모니터링 시스템 및 웹 대시보드 프로젝트의 누적 개발 이력과 마일스톤을 기록하는 타임라인 문서입니다.

---

## 📅 프로젝트 히스토리 요약

### 1. [완료] 오늘 이전 (초기 구축 및 서비스 완성품 단계)
*   **실시간 데이터 업로더 개발**
    - SQLite 로컬 DB에서 신규 측정 데이터를 수집하여 클라우드로 전송하는 파이썬 업로더(`uploader.py`, `gui_uploader.py`) 엔진 구축.
    - Supabase PostgreSQL 적재 및 구글 스프레드시트 일괄 연동 기능 구현.
    - 로컬 테스트 환경을 위한 CSV 모의 업로드 모드(Mock Mode) 탑재.
*   **웹 모니터링 대시보드 구축 (React + Vite)**
    - 계측 데이터의 실시간 동기화 및 Recharts 기반의 `TOC & Signal Multi-Series Trend` 시각화 차트 개발.
    - 모니터링 보안을 위한 관리자 접속 패스코드(Passcode: `850`) 검증 관문 적용.
    - 날짜별 점프(검색), 채널 필터링, TOC 범위별 그리드 데이터 테이블 및 CSV 데이터 다운로드 모달 기능 구현.
    - 다크모드 기반의 글래스모피즘(Glassmorphism) 프리미엄 UI 스타일 시트 정비.

### 2. [진행 중] 오늘 (2026년 6월 18일)
*   **[완료] 트렌드 차트 표시범위 최적화**
    - `TOC & Signal Multi-Series Trend`의 첫 페이지 로드 시 표시범위 기본값을 기존 7일(`7d`)에서 **24시간(`24h`)**으로 축소 변경하여 모니터링 로딩 속도 향상 및 사용성을 보정했습니다.
*   **[진행 및 검토] 3단계 알림 고도화 및 이메일 전송 연동 검토**
    - **요청 사항**: 평시(흰색), 주의(노랑), 경고(빨강)의 3단계 등급 시각화 구현 및 채널별 커스텀 임계값 지정 요구 수렴.
    - **채널별 요구 사양**: 
      - Ch 3 (방류수): 주의 40 / 경고 50 (ppm)
      - Ch 2 (고농도조): 주의 900 / 경고 1000 (ppm)
      - Ch 1 (원수조): 주의 1600 / 경고 2000 (ppm)
    - **이메일 연동**: 경고 기준치 초과 시 ecounterp SMTP 서버를 경유하여 관리자 주소록으로 실시간 이메일 경보를 발송하는 로직 및 알림 폭주 방지(1시간 쿨다운) 적용 검토 중.
*   **[진행 및 검토] 독립 관리자 테스트 전용 웹페이지(?admin) 아키텍처 검토**
    - 일반 사용자용 메인 모니터링 화면을 침범하지 않고 검증하기 위해 `?admin` 주소를 활용한 전용 모의 테스트 샌드박스(TOC 데이터 적재 시뮬레이터 및 메일 발송 비동기 신호 트리거) 구현안 설계 중.
*   **[완료] 협업 기반 구축**
    - 다른 채팅 세션이나 에이전트 간 업무 인수인계 및 맥락 보존을 위해 `meeting_notes` 폴더 신설 및 회의록 파일(`2026-06-18_alert_and_test_page_design.md`) 보관 완료.
