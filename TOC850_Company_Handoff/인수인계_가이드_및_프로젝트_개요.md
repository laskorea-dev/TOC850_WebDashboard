## 이 시스템은 뭘 하는 건가요?

현장에 설치된 TOC850 수질 계측기가 측정한 데이터를 **인터넷 클라우드(Supabase)**에 자동으로 올려주고, 고객사 담당자가 **웹 브라우저(PC·스마트폰)로 어디서든 실시간 확인**할 수 있는 대시보드를 제공하는 시스템입니다.

---

## 전체 구조 

```
[현장 PC]                    [인터넷 클라우드]                [고객사 열람]
                                                            
계측기(TOC850)               Supabase                       웹 브라우저
    ↓                        (PostgreSQL DB)                    ↑
toc_db.db (SQLite)  ──→  gui_uploader.exe  ──→  데이터 저장  ──→  대시보드 웹
(자동 생성 DB 파일)     (설정에 따라 자동 전송)                   (vercel.app 주소)
```

### 핵심 흐름 3단계
1. **계측기** → 현장 PC에 `toc_db.db` 파일로 측정 데이터 자동 저장
2. **업로더 프로그램** (`gui_uploader.exe`) → `uploader_config.json` 파일에 설정된 시간마다 `toc_db.db`의 신규 데이터를 읽어서 인터넷(Supabase 클라우드 DB)으로 전송
3. **웹 대시보드** → Supabase에 저장된 데이터를 실시간으로 읽어와 차트·표로 보여줌 (고객사는 발급받은 URL로 접속)

## 현장 운영: 자주 발생하는 상황별 대응법

### 🔹 상황 1: 대시보드 웹 화면이 안 열려요
- **원인**: Vercel 호스팅 서비스 문제이거나, Supabase 무료 티어 일시 정지
- **대응**: 
  1. Vercel(https://vercel.com) 에 로그인해서 프로젝트가 정상인지 확인
  2. Supabase(https://supabase.com/dashboard) 에 로그인해서 프로젝트가 "Active" 상태인지 확인
  3. Supabase 무료 티어는 **7일간 비활성 시 일시 정지(Pause)**될 수 있음 → 대시보드에서 "Resume" 버튼 클릭

### 🔹 상황 2: 데이터가 더 이상 올라오지 않아요
- **원인**: 현장 PC의 업로더 프로그램이 꺼졌거나, 인터넷 연결 문제
- **대응**:
  1. 현장 PC에서 `gui_uploader.exe`가 실행 중인지 확인
  2. 프로그램 창의 로그 영역에 에러 메시지가 있는지 확인
  3. PC의 인터넷 연결 상태 확인

### 🔹 상황 3: 새로운 고객사에 대시보드를 추가 제공해야 해요
- **요약**: Supabase에 새 테이블 생성 → Vercel에 새 프로젝트 배포 (환경변수만 다르게 설정)
- **상세 절차**: `4_웹_대시보드_무료_배포_가이드.md` 문서의 "B2B 멀티테넌시" 섹션 참조
- ⚠️ 이 작업은 **개발 지식이 필요**하므로 외주를 권장합니다.

### 🔹 상황 4: 대시보드 디자인이나 기능을 수정해야 해요
- **대응**: `dashboard/src/App.jsx` 파일을 수정 → `npm run build` → Vercel 재배포
- ⚠️ 이 작업은 **React/JavaScript 개발 지식이 필요**하므로 외주를 권장합니다.

## Supabase 클라우드 DB 구조

### 현재 사용 중인 테이블

데이터는 **고객사(사이트)별로 테이블을 따로 만들어** 물리적으로 분리합니다.

| 테이블 이름 | 설명 |
|------------|------|
| `Samyang_Incheon` | 현재 운영 중인 고객사 데이터 테이블 (예시) |
| `850_dashboard_site_config` | 사이트별 설정 정보 (사이트명, 보안코드, 알림 임계값 등) |
| `measure_logs` | 업로더 기본 전송 대상 테이블 |

### 테이블 컬럼 구조 (measure_logs / 고객사 테이블)

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `Date_Time` | text | 측정 일시 (YYYY-MM-DD HH:MM:SS) |
| `Device_ID` | text | 계측 장비 고유 식별자 (예: DEVICE_01) |
| `Channel` | integer | 측정 채널 번호 (1, 2, 3, 4...) |
| `Channel_Name` | text | 채널 이름 (방류수, 유입수, 고농도 등) |
| `TOC_Conc` | float | TOC 농도 측정값 (핵심 데이터) |
| `DilutionFactor` | float | 희석 배율 |
| `MSIG` | float | 측정 신호값 |
| `SLOP` | float | 검량선 기울기 |
| `ICPT` | float | 검량선 절편 |
| `FACT` | float | 보정 계수 |
| `OFST` | float | 오프셋 값 |
| `MAXR` | integer | 최대 반복 횟수 |
| `Add_note` | text | 비고 / 추가 정보 |

### site_config 테이블 (850_dashboard_site_config)

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `site_id` | text | 사이트 식별자 (= 데이터 테이블명과 동일) |
| `passcode` | text | 대시보드 접속 보안 코드 |
| `site_name` | text | 대시보드에 표시될 사이트 이름 |
| `is_active` | boolean | 서비스 활성화 여부 (false이면 접근 차단) |
| `toc_alert_high` | jsonb | 채널별 TOC 경고 임계값 (JSON 형태) |

## 대시보드 접속 방식 (URL 구조)

웹 대시보드는 URL의 `?site=` 파라미터로 고객사를 구분합니다.

```
https://배포주소.vercel.app/?site=Samyang_Incheon
                                   ↑
                            Supabase 테이블 이름과 동일
```

- `?site=` 파라미터가 없으면 → "접속 제한" 안내 화면 표시
- `?site=` 파라미터가 있으면 → 해당 테이블의 데이터를 보여주는 대시보드 표시
- 접속 시 **보안 코드(passcode)** 입력 필요 (site_config 테이블에서 관리)

## 업로더 프로그램 설정 (uploader_config.json)

업로더 프로그램의 동작을 제어하는 설정 파일입니다. GUI 프로그램 창에서도 수정할 수 있지만, 직접 편집도 가능합니다.

```json
{
    "db_path": "toc_db.db 파일의 전체 경로",
    "supabase_table": "measure_logs",
    "device_id": "DEVICE_01",
    "interval_seconds": 900,
    "is_mock": false,
    "supabase_url": "https://xxxx.supabase.co/rest/v1/",
    "supabase_key": "Supabase API 키 값"
}
```

| 설정 | 설명 |
|------|------|
| `db_path` | 현장 PC의 SQLite DB 파일 위치 |
| `supabase_url` | Supabase 프로젝트의 REST API 주소 |
| `supabase_key` | Supabase의 API 인증 키 (service_role 키) |
| `supabase_table` | 데이터를 넣을 테이블 이름 |
| `device_id` | 이 장비의 고유 식별자 (여러 대 구분용) |
| `interval_seconds` | 자동 전송 주기 (초 단위, 기본 900 = 15분) |
| `is_mock` | true이면 클라우드 전송 없이 로컬 CSV로 시뮬레이션 |

## 대시보드 환경변수 (.env 파일)

웹 대시보드가 Supabase에 접속하기 위한 설정입니다. `dashboard/.env.local` 파일 또는 Vercel 환경변수로 관리합니다.

```
VITE_SUPABASE_URL=https://xxxx.supabase.co/rest/v1/
VITE_SUPABASE_KEY=Supabase API 키 값
VITE_SUPABASE_TABLE=Samyang_Incheon
```

- **로컬 개발 시**: `dashboard/.env.local` 파일에 직접 기입
- **Vercel 배포 시**: Vercel Dashboard → Settings → Environment Variables에 등록 (소스코드에 키가 노출되지 않도록)

## 계정 이전 방법 (회사 소유로 전환)
